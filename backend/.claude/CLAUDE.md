# Ezo POS — Backend API
> Node.js + Express + TypeScript | ORM: Drizzle | DB: PostgreSQL | Auth: JWT + Google OAuth

## Stack
- Express 4 + TypeScript 5.7
- Drizzle ORM — schema at src/db/schema/index.ts, migrations at ./migrations/
- Zod for ALL input validation — no request body is ever trusted without Zod parse
- JWT for session auth, Google OAuth for sign-in
- RBAC middleware in src/middleware/rbac.ts
- Helmet + CORS + Morgan already wired in

## Folder structure rules
src/
  routes/        — register router only, no logic (thin)
  controllers/   — handle req/res, call service, return response
  services/      — ALL business logic here, no req/res objects
  repositories/  — ALL Drizzle queries here, services call repositories
  middleware/    — auth, rbac, validation, error handling
  db/
    schema/      — Drizzle table definitions (index.ts re-exports all)
    migrations/  — generated by drizzle-kit
  validators/    — Zod schemas, imported by controllers

## The sync endpoint contract
Flutter syncs by uuid (never by integer id). All sync endpoints:
- Accept arrays of objects with uuid field
- Use INSERT ... ON CONFLICT (uuid) DO UPDATE (upsert)
- Return { synced: uuid[], failed: uuid[], timestamp: ISO }
- Support ?since=ISO query param for delta sync (updatedAt > since)

## Response format (always)
Success: { success: true, data: T, meta?: {} }
Error:   { success: false, error: { code: string, message: string } }
Never return raw Drizzle objects — always map through a presenter/serializer

## Security non-negotiables
- drizzle.config.ts MUST read from process.env, never hardcoded credentials
- All routes except /auth/* require verifyToken middleware
- RBAC roles: admin | manager | cashier — defined in src/types/roles.ts
- Rate limiting on /auth/* routes

## Migrations
Run: npm run db:migrate
